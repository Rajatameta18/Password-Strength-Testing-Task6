# 🚨 Common Password Attacks

## 1. Brute Force Attack

A brute-force attack tries every possible password combination until the correct password is found.

---

## 2. Dictionary Attack

A dictionary attack uses a predefined list of common passwords and dictionary words to guess passwords.

Examples:
- password
- 123456
- admin
- welcome123

---

## 3. Credential Stuffing

Attackers use usernames and passwords leaked from previous data breaches to gain unauthorized access to accounts.

---

## 4. Phishing

Users are tricked into entering their passwords on fake websites or through fraudulent emails.

---

#  Best Practices for Creating Strong Passwords

- Use at least **12–16 characters**.
- Include uppercase and lowercase letters.
- Include numbers and special symbols.
- Avoid dictionary words.
- Avoid personal information like your name or birth year.
- Use a unique password for every account.
- Use passphrases instead of short passwords.
- Enable Multi-Factor Authentication (MFA).
- Store passwords securely using a password manager.

---

#  What I Learned

- Password length significantly improves security.
- Using multiple character types increases password complexity.
- Dictionary words and common patterns make passwords weak.
- Personal information such as names or birth years should be avoided.
- Password managers help generate and securely store strong passwords.
- Multi-Factor Authentication provides an extra layer of protection.
